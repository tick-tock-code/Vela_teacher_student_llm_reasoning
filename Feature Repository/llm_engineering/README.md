# LLM Engineering Features

17 binary features generated by an LLM reasoning pipeline. Unlike the
Policy Induction features (which are LLM *policies* scored against founders),
these are LLM-generated **feature definitions** — the LLM was prompted to
invent what to measure, and the resulting feature values were computed by
the same LLM examining each founder's profile.

## What's in here

| File | Shape | Description |
|------|-------|-------------|
| `features.csv` | (17, 10) | Metadata for the 17 features: id, name, llm, fire rate, univariate F0.5 |
| `features_train.csv` | (4500, 18) | `founder_uuid` + 17 binary features. **100 founders have NaN** (see below). |
| `features_test.csv`  | (4500, 18) | `founder_uuid` + 17 binary features. Fully populated. |
| `seed_uuids.txt` | — | The 100 training founders with NaN features (reserved as LLM seed examples) |

Feature IDs are all prefixed with `le_` (LLM Engineered) to distinguish them
from the Policy Induction features (`v25_`) and RRF questions (`rrf_Q`).

## The 100-founder seed holdout

The LLM-engineered features were produced by prompting an LLM with 100 seed
example founders (drawn randomly from the training set with `random_state=42`),
asking it to propose features, then scoring the remaining 4,400 training
founders and all 4,500 test founders.

The 100 seed founders are therefore **not scored** with these features (they
were used to *create* the features, and scoring them would be a leak). They
appear in `features_train.csv` with NaN values for all 17 feature columns.

When modelling with LLM-engineered features, drop the seed rows:

```python
le_train = pd.read_csv("features_train.csv")
has_le = le_train.iloc[:, 1].notna()  # any le_ column works
train_4400 = le_train[has_le]          # the 4400 founders that can be modelled
```

`seed_uuids.txt` contains the 100 UUIDs explicitly so you can drop them from
other data sources too.

## Performance (univariate F0.5 on 4400 training founders)

See `features.csv` for per-feature metrics. The strongest individual
features typically fire 20-40% of the time on training founders and have
pos lift around 1.2-2.0.

As an **ensemble**, these 17 features alone achieve:
- Test F0.5 = **0.284** (nested L2 LR, no rule override)
- CV F0.5 ≈ 0.266 (5-fold OOF)

When combined with Policy Induction (16 v25 policies), the result is the
strongest feature combination in the repository:
- **Test F0.5 = 0.334** (LLM-Eng + Policy Induction, 33 total features)

## Important notes

- **These features are NOT in the semantic similarity index** (only Policy
  Induction and RRF questions are indexed). If you want to look up whether
  a feature idea already exists here, search `features.csv` by text match.
- **Do NOT standardise**: these features are binary. The final experiments
  leave them unstandardised. If you're building a pipeline that standardises
  continuous features, add `le_*` to your binary feature exclusion list.
- The features are **integer-typed** (0/1). Some sklearn models might
  require casting to float.
