# Policies

LLM-derived feature sets where each "feature" is a natural-language statement
about a founder, scored by an LLM against each founder's profile. There are
two sources, with different LLMs, prompt formats, and scoring conventions.

## What's in here

| File | Shape | Description |
|------|-------|-------------|
| `policies.csv` | (36, 17) | Metadata for all 36 policies: feature_id, source, text, llm, fire rates, metrics, ensemble coefficients |
| `predictions_train.csv` | (4500, 37) | `founder_uuid` + 36 policy columns. Aligned with `../splits/train_uuids.txt`. |
| `predictions_test.csv` | (4500, 37) | Same shape, for the test set. Aligned with `../splits/test_uuids.txt`. |
| `prompts/policy_induction_v25.md` | — | Prompt template for the v25 policies |
| `prompts/taste_policy_decomposition.md` | — | Prompt template for the Taste policies |

## The two sources

### `policy_induction_v25` — 16 policies (Gemini 2.5 Flash, probability scoring)

The v25 set is a 16-policy ensemble selected from a larger pool of 200+
candidates generated via policy induction. Selection was done with bootstrap
significance testing on a held-out subset.

- **Feature IDs**: `v25_p1`, `v25_p11`, `v25_p38`, `v25_p52`, `v25_p55`,
  `v25_p58`, `v25_p72`, `v25_p80`, `v25_p112`, `v25_p116`, `v25_p121`,
  `v25_p135`, `v25_p143`, `v25_p150`, `v25_p157`, `v25_p161`
- **Scoring format**: probability in [0, 1]. Each value is `P(success | policy
  applies, founder)`, computed from next-token logprobs over True/False.
- **LLM**: gemini-2.5-flash, temperature=0
- **Coverage**: train (4500) + test (4500), both fully scored
- **Best ensemble result**: **CV F0.5 = 0.294** (5-fold L2 LogReg, OOF threshold tuning)

Source experiment: `experiments/policy_induction_vcbench/results/v25_*`. The
policy text was originally generated by an LLM and then curated by hand. The
final 16 policies cover macro-market signals, founder background red flags,
domain expertise, and exit history.

### `taste_policy_decomposition` — 20 policies (Gemini 2.0 Flash, binary scoring)

20 hand-crafted policies decomposed from a "taste profile" containing 278 real
investment decisions distilled into patterns. The policies are organised into
three categories:

- **Accept signals (A1-A8)**: positive indicators (prior IPO, top PhD,
  C-level role at large company, etc.)
- **Reject signals (R1-R7)**: negative indicators (long tenure no
  progression, back-office roles, consulting/agency career, etc.)
- **Meta signals (M1-M5)**: secondary indicators (elite university, 10+
  years experience, fast career velocity, etc.)

- **Feature IDs**: `taste_A1_prior_acquisition`, ..., `taste_M5_very_large_company_exp`
- **Scoring format**: binary 0/1 (converted from YES/NO)
- **LLM**: gemini-2.0-flash, temperature=0
- **Coverage**: train (public 4500) + test (private 4500), both fully scored
- **Best ensemble result**: CV F0.5 = 0.308 (10-fold L2 LogReg)

Source experiment: `experiments/taste_policy_decomposition/`. The policy
definitions are inline in `run_taste_policies.py`.

## Important notes

- **Different scoring formats**. v25 features are probabilities in [0, 1];
  Taste features are binary 0/1. If you stack them in a single LR/XGB model,
  consider whether you want to threshold the v25 features to binary first
  (typically at 0.5) — though both work fine in tree-based models.

- **Founder UUIDs are aligned to the canonical splits.** Both
  `predictions_train.csv` and `predictions_test.csv` are line-aligned with
  `../splits/{train,test}_uuids.txt`. You can join freely on `founder_uuid`.

- **No NaN values in the prediction matrices.** Both sources have full
  coverage. The Taste features were rebuilt by parsing YES/NO strings from
  the source CSVs; missing or malformed entries became 0.

- **Ensemble coefficients in `policies.csv`** come from the final 16-policy
  L2 LogReg model selected via nested CV. They reflect each policy's marginal
  contribution conditional on all other v25 policies. The Taste policies do
  not have ensemble coefficients in this file (the 20-policy LR was fit on
  Taste features alone — those coefficients live in
  `experiments/taste_policy_decomposition/results/lr_holdout_results.json`).

## Quick load example

```python
import pandas as pd

policies = pd.read_csv("policies.csv")
train = pd.read_csv("predictions_train.csv")
test = pd.read_csv("predictions_test.csv")

# Get just the v25 policies
v25_cols = [c for c in train.columns if c.startswith("v25_")]
X_v25_train = train[["founder_uuid"] + v25_cols]

# Get just the Taste policies  
taste_cols = [c for c in train.columns if c.startswith("taste_")]
X_taste_train = train[["founder_uuid"] + taste_cols]

# Reproduce the v25 CV F0.5 ≈ 0.294 result
import numpy as np
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import fbeta_score
from sklearn.model_selection import StratifiedKFold

labels = pd.read_csv("../splits/labels.csv")
y = labels[labels.split == "train"].set_index("founder_uuid").loc[
    train["founder_uuid"]
]["success"].values

X = train[v25_cols].values
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
oof = np.full(len(y), np.nan)
for ti, vi in skf.split(X, y):
    m = LogisticRegression(C=0.005, penalty="l2", max_iter=2000)
    m.fit(X[ti], y[ti])
    oof[vi] = m.predict_proba(X[vi])[:, 1]

best_f = max(
    fbeta_score(y, (oof >= t).astype(int), beta=0.5, zero_division=0)
    for t in np.arange(0.05, 0.95, 0.01)
)
print(f"v25 CV F0.5 = {best_f:.4f}")  # ~0.294
```
